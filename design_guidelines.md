# ERP 시스템 UI/UX 디자인 가이드라인

본 문서는 ERP 시스템의 일관된 사용자 경험과 효율적인 업무 처리를 위한 디자인 원칙 및 가이드를 정의합니다.

## 1. 디자인 원칙 (Design Principles)

*   **명확성 (Clarity)**: 복잡한 데이터를 다루는 ERP 특성상, 정보의 가독성을 최우선으로 합니다. 불필요한 장식을 배제하고 데이터 자체에 집중할 수 있는 디자인을 지향합니다.
*   **효율성 (Efficiency)**: 사용자의 업무 동선을 최소화하고, 반복적인 작업(실적 입력 등)을 신속하게 처리할 수 있도록 직관적인 UI를 제공합니다.
*   **일관성 (Consistency)**: 모든 화면에서 동일한 레이아웃, 색상, 폰트, 인터랙션 규칙을 적용하여 사용자의 학습 비용을 낮춥니다.

## 2. 레이아웃 (Layout)

전체적인 레이아웃은 **좌측 사이드바 네비게이션(LNB)** 구조를 기본으로 합니다.

*   **GNB (Global Navigation Bar)**: 상단 영역. 로고, 검색, 알림, 사용자 프로필, 설정 메뉴 배치.
*   **LNB (Local Navigation Bar)**: 좌측 영역. 대메뉴(계획, 실적, 리포트 등) 및 하위 메뉴 트리 구조. 접기/펼치기 기능 지원.
*   **Content Area**: 중앙 영역. 실제 업무 화면이 표시되는 공간. Breadcrumb(경로 표시)를 상단에 배치하여 현재 위치 파악 용이.

## 3. 컬러 팔레트 (Color Palette)

신뢰감을 주는 블루 계열을 메인 컬러로 사용하고, 상태 표시를 위한 시맨틱 컬러를 정의합니다.

### Primary Colors
*   **Main Blue**: `#0052CC` (브랜드 아이덴티티, 주요 버튼, 활성 상태)
*   **Light Blue**: `#E3F2FD` (배경, 호버 상태, 보조 강조)
*   **Dark Navy**: `#172B4D` (헤더, LNB 배경, 강한 텍스트)

### Semantic Colors (Status)
*   **Success (완료/달성)**: `#36B37E` (KPI 달성, 승인 완료)
*   **Warning (지연/검토)**: `#FFAB00` (일정 지연, 결재 대기)
*   **Danger (미달성/반려)**: `#FF5630` (KPI 미달성, 반려, 삭제)
*   **Info (진행중)**: `#00B8D9` (진행 중인 태스크)

### Neutral Colors
*   **Text Black**: `#172B4D` (본문 텍스트)
*   **Text Grey**: `#6B778C` (보조 텍스트, 라벨)
*   **Border**: `#DFE1E6` (테두리, 구분선)
*   **Background**: `#F4F5F7` (페이지 배경)

## 4. 타이포그래피 (Typography)

가독성이 뛰어난 산세리프(San-serif) 서체를 사용합니다. (예: Pretendard, Noto Sans KR)

*   **Headings**:
    *   H1 (페이지 타이틀): 24px, Bold
    *   H2 (섹션 타이틀): 20px, Semi-Bold
    *   H3 (카드 타이틀): 16px, Medium
*   **Body**:
    *   Default: 14px, Regular (본문, 테이블 데이터)
    *   Small: 12px, Regular (도움말, 메타 정보)

## 5. 주요 UI 컴포넌트 (Components)

### 5.1 데이터 테이블 (Data Grid)
*   많은 양의 데이터를 효율적으로 보여주기 위해 **페이지네이션**, **필터링**, **정렬** 기능을 기본 제공합니다.
*   행(Row) 호버 시 배경색 변경으로 가독성을 높입니다.
*   중요한 숫자(금액, 달성률)는 우측 정렬하고, 상태값은 배지(Badge) 형태로 시각화합니다.

### 5.2 대시보드 카드 (Dashboard Card)
*   핵심 지표(KPI)를 보여주는 카드 UI입니다.
*   타이틀, 주요 수치(크게), 전월 대비 증감률(화살표 아이콘 + 색상), 미니 차트(Sparkline)로 구성합니다.

### 5.3 입력 폼 (Input Form)
*   라벨은 입력 필드 상단에 배치하여 시선의 흐름을 자연스럽게 합니다.
*   필수 입력 항목은 `*` 표시로 명확히 구분합니다.
*   복잡한 입력(연간 계획 등)은 단계별 위자드(Wizard) 또는 탭(Tab) UI를 활용하여 입력을 분산시킵니다.

### 5.4 캘린더 뷰 (Calendar View)
*   월간/주간/일간 뷰를 제공합니다.
*   일정 클릭 시 상세 팝업(Modal)을 띄워 내용을 확인하고 실적으로 등록할 수 있게 합니다.
*   Google Calendar 연동 일정은 별도 아이콘으로 표시하여 구분합니다.

## 6. 인터랙션 (Interaction)

*   **피드백**: 버튼 클릭, 저장 등 사용자 액션에 대해 즉각적인 피드백(Toast Message, Spinner)을 제공합니다.
*   **모달(Modal)**: 상세 정보 조회나 간단한 입력은 페이지 이동 없이 모달 팝업을 활용하여 맥락을 유지합니다.
*   **차트**: 마우스 호버 시 툴팁(Tooltip)을 통해 상세 수치를 보여줍니다.
